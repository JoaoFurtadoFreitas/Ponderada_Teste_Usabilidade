# Conjunto de perguntas utilizando a técnica do funil para testar um aspecto do projeto

## 1. Tela do Dashboard
A tela escolhida para o teste foi a tela inicial (dashboard), responsável por concentrar a maior parte das informações do sistema.



---

## 2. Tipo de Teste
O teste realizado é do tipo **visualização**, abordando os seguintes critérios:
- Compreensão
- Explicabilidade
- Efeito
- Praticabilidade

---

## 3. Conjunto de Perguntas (Técnica do Funil)

### Perguntas gerais (visão macro)
- O usuário consegue visualizar o fluxo de recursos no mapa?

### Perguntas intermediárias (interpretação)
- O usuário consegue relacionar o fluxo de recursos com a alocação para uma ocorrência?
- O usuário compreende a relação entre rotas, ocorrências e recursos apresentados?

### Perguntas específicas (interação e detalhe)
- O usuário consegue identificar os valores de cada tipo de recurso ao interagir com os elementos do mapa (linhas de rota e círculos das ocorrências)?
- O usuário consegue filtrar o mapa por uma ocorrência específica?
- Após o filtro, o usuário consegue:
  - Aguardar o carregamento sem dificuldade?
  - Interagir com os elementos (clusters de rastreadores, rotas e ocorrências)?
  - Visualizar os dados apresentados?
  - Relacionar essas informações com:
    - A alocação de recursos
    - O estado da ocorrência
    - O algoritmo utilizado?

---

## 4. Objetivo do Teste
O objetivo é avaliar o nível de intuitividade dos elementos visuais do mapa e a clareza das informações apresentadas ao longo da interação do usuário com o dashboard.

---

## 5. Ação ou Entendimento Esperado
Espera-se que o usuário seja capaz de associar corretamente os elementos visuais do mapa ao fluxo de alocação de recursos para as ocorrências, compreendendo o funcionamento geral do sistema.
